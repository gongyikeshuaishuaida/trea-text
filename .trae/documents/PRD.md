# 学习平台产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品名称
《共享单车都去哪了？——编程处理数据》互动学习平台

### 1.2 产品定位
基于高中信息技术学考复习课程《4.2.2 编程处理数据》的互动式在线学习平台，通过真实情境模拟和渐进式代码练习，帮助学生掌握pandas数据处理技能。

### 1.3 目标用户
- **主要用户**：浙江高中信息技术学考学生
- **次要用户**：信息技术教师

### 1.4 核心价值
- 将抽象的数据处理概念转化为可视化、可交互的学习体验
- 通过情境模拟激发学生探究兴趣
- 提供即时反馈的代码练习强化学习效果

## 2. 教学背景

### 2.1 课程信息
| 项目 | 内容 |
|------|------|
| 课题 | 4.2.2 编程处理数据 |
| 课型 | 学考复习课 |
| 课时 | 40分钟 |
| 适用对象 | 浙江高中信息技术学考学生 |

### 2.2 教学目标
- **知识与技能**：掌握pandas数据读取、筛选、分组统计、排序、matplotlib折线图绘制
- **过程与方法**：掌握"读，筛，算，排，画"的数据处理方式
- **情感态度**：感受数据分析解决现实问题的价值，培养数据思维

### 2.3 数据文件说明
**文件名**：bike_data.csv

| 字段名 | 数据类型 | 说明 | 示例 |
|--------|----------|------|------|
| 时间 | 字符串 | 10分钟为间隔 | "7:10", "7:20" |
| 星期 | 字符串 | 周几 | "周六", "周日" |
| 区域类型 | 字符串 | 功能区划分 | "住宅区", "公交站", "商圈" |
| 位置名称 | 字符串 | 具体地点 | "某小区西门", "公交站A西侧" |
| 车辆数 | 整数 | 当前停放单车数 | 92 |

**数据规模**：3种区域 × 9个具体位置 × 78个时刻 × 1天

**数据规律**：早高峰车在住宅区，午高峰车变少，晚高峰车回流住宅区

## 3. 功能需求

### 3.1 首页/登录页
- 简洁的登录界面，支持学生账号登录
- 显示课程名称和基本信息
- 醒目的"进入学习"按钮

### 3.2 情境再现模块 ⭐
**功能描述**：交互式模拟共享单车场景，让学生观察单车流动规律

**交互设计**：
- 可视化展示某小区门口的共享单车
- 时间控制按钮：「-1小时」「-30分钟」「-10分钟」「+10分钟」「+30分钟」「+1小时」
- 实时显示当前时间和对应地点的车辆数
- 动画效果展示单车数量变化

**初始状态（7:00）**：
| 地点 | 车辆数 |
|------|--------|
| 某小区西门 | 41 |
| 某小区东门 | 43 |
| 某小区南门 | 45 |

**观察任务**：
- 记录：几点开始小区门口的车明显减少？
- 预期答案：8:30左右

### 3.3 知识库浮窗 📖
**功能描述**：悬浮的知识参考窗口，随时可查阅pandas基础知识

**触发方式**：
- 页面右上角固定「📖知识库」按钮
- 点击展开浮窗，可拖拽
- 点击浮窗外部或关闭按钮收起

**知识内容**：
1. **pandas读取数据**：`pd.read_csv("文件名.csv")`
2. **数据筛选**：`df[ df["列名"] == 条件 ]`
3. **分组统计**：`df.groupby("列名").agg({"列名": 聚合函数})`
4. **排序**：`df.sort_values("列名", ascending=True/False)`
5. **matplotlib绑定图**：`plt.bar(x数据, y数据)`

### 3.4 活动一：设计分析流程
**功能描述**：拖拽排序练习，让学生理解数据分析的步骤

**步骤选项**：
- A. 查看各个区域的车辆数据
- B. 查看哪个区域的车最少，哪个区域的车最多
- C. 根据车消失的时间，确定对哪个时间的车辆数据进行分析
- D. 可视化呈现

**正确答案**：C → A → B → D

**交互设计**：
- 4个可拖拽的步骤卡片
- 拖拽排序后点击"检查答案"
- 正确时显示绿色勾选和鼓励语
- 错误时高亮错误位置并提示

### 3.5 活动二：筛选数据
**功能描述**：填空练习，掌握数据筛选语法

**代码模板**：
```python
import pandas as pd
df = pd.read_csv("bike_data.csv")
# 筛选：从所有时刻记录里，找出8:30的所有行
timebike = df[ df["时间"] _________  ]
print(timebike)
```

**填空答案**：`== "8:30"`

**交互设计**：
- 代码高亮显示
- 下划线位置可输入
- 点击"运行代码"查看结果
- 显示筛选后的数据表格

### 3.6 活动三：分组统计
**功能描述**：两步填空练习，掌握groupby分组聚合

**第一阶段代码**：
```python
# 按区域类型分组，求车辆数的总和
df1 = timebike.groupby(_____).______
```

**填空答案**：`"区域类型"`, `sum()`

**观察问题**：发现什么问题？（字符串拼接错误）

**第二阶段代码**：
```python
# 按区域类型分组，只对车辆数求和
df1 = timebike.groupby("_____", as_index=False)_____["_____"]._____
```

**填空答案**：`"区域类型"`, `"车辆数"`, `sum()`

### 3.7 活动四：排序
**功能描述**：填空练习，掌握sort_values排序方法

**代码模板**：
```python
# 按车辆数降序排列，找到车最多和最少的地方
df2 = df1.sort_values("车辆数", ascending=_______)
print("车最多的区域为：")
print(df2.head(1))
print("车最少的区域为：")
print(df2.tail(1))
```

**填空答案**：`False`

**结果展示**：
- 早上8:30，**商圈/公交站**的车最多，**住宅区**的车最少

### 3.8 活动五：可视化
**功能描述**：填空练习，掌握matplotlib柱状图绘制

**代码模板**：
```python
import matplotlib.pyplot as plt
plt.rcParams["font.sans-serif"] = ["SimHei"]

# 画柱形图
plt.bar(_____, _____)
plt.title("8:30各区域共享单车数量对比")
plt.xlabel("区域类型")
plt.ylabel("车辆总数")
plt.show()
```

**填空答案**：`区域类型数据`, `车辆数数据`

**交互设计**：
- 代码填写完整后点击"绘图"
- 显示生成的柱状图
- 可切换查看不同时间点的图表

### 3.9 数据驱动决策展示
**功能描述**：总结页面，展示数据分析结论的实际应用

**内容**：
| 场景 | 数据依据 | 决策行动 |
|------|----------|----------|
| 早高峰前 | 7:00住宅区车多 | 无需调度，保持现状 |
| 早高峰时 | 8:30车流向商圈/公交站 | 从住宅区调车到商圈/公交站 |
| 晚高峰前 | 数据预测 | 提前从商圈调车回住宅区 |

### 3.10 课堂小结
**功能描述**：课程知识点总结

**内容**：
- 核心方法论："读数据 → 筛数据 → 算数据 → 排数据 → 画数据"
- 强调：用编程处理数据，用数据驱动决策，解决生活中的问题
- 延伸应用：电商推荐、交通信号灯、超市货架等
